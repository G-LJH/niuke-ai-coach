# 安全与运行规则

## 文件访问
- 只能读写 `./data/` 目录及其子目录
- 禁止访问系统目录和上级目录
- 上传文件先存入 `./data/uploads/`，处理完成后可清理

## 网络请求
- 允许域名：`nowcoder.com`、`dashscope.aliyuncs.com`
- 其他域名一律禁止
- 所有网络请求必须设置超时：连接 10s，读取 30s

## 速率限制
- 牛客网请求间隔 ≥ 1 秒
- 连续失败后等待 60 秒重试，最多 3 次
- 百炼 API 调用频率 ≤ 10 次/分钟

## 密钥管理
- 从 `.env` 文件读取，不写入代码
- `.env` 不提交到仓库
- 环境变量名：`DASHSCOPE_API_KEY`

## 日志规范

### 日志文件
- 位置：`./data/logs/`
- 命名：`app_{YYYY-MM-DD}.log`
- 轮转：按天轮转，保留 7 天

### 日志格式
JSON 格式，每行一条：
```json
{"timestamp": "ISO时间", "level": "INFO", "module": "workflow", "message": "工作流启动", "data": {...}}
```

### 日志级别
| 级别 | 说明 | 示例 |
|------|------|------|
| `DEBUG` | 调试信息 | 工具调用参数、响应详情 |
| `INFO` | 正常流程 | 工作流启动/完成、步骤切换 |
| `WARN` | 警告信息 | 面经抓取成功率低、数据不足 |
| `ERROR` | 错误信息 | 工具调用失败、API 异常 |

### 必须记录的内容
- **工具调用**：工具名、耗时（ms）、成功/失败、错误信息（如有）
- **工作流**：工作流ID、步骤名、状态变更
- **API 调用**：模型名、输入/输出 token 数、耗时

```json
{"timestamp": "2025-04-18T10:00:00Z", "level": "INFO", "module": "tool", "tool": "fetch_interview_content", "duration_ms": 1200, "status": "success"}
{"timestamp": "2025-04-18T10:00:01Z", "level": "INFO", "module": "workflow", "workflow_id": "wf_001", "step": 2, "step_name": "fetch_interview_exps", "status": "running"}
```

## 错误处理

### 错误分类
| 错误类型 | 处理策略 | 示例 |
|----------|----------|------|
| 网络超时 | 重试 3 次，间隔递增（10s, 30s, 60s） | 面经抓取超时 |
| API 限流 | 等待 `retry_after` 秒后重试 | 百炼 API 429 |
| 文件解析失败 | 终止流程，返回明确错误 | PDF 损坏 |
| 数据不足 | 记录警告，降级继续执行 | 面经抓取成功率 < 50% |
| 模型输出异常 | 重试生成，最多 2 次 | JSON 解析失败 |

### 降级策略
1. **面经数据不足**：成功率 < 50% 时，在报告中添加提示 "面经数据较少，建议补充"
2. **JD 缺失**：无 JD 输入时，仅基于岗位名称和简历生成报告，标注 "缺少岗位描述，分析可能不够精准"
3. **部分模块生成失败**：已生成模块正常输出，失败模块显示 "该模块生成失败，请稍后重试"

### 错误响应格式
```json
{
  "success": false,
  "error": {
    "code": "FETCH_TIMEOUT",
    "message": "面经抓取超时，已重试 3 次",
    "step": 2,
    "step_name": "fetch_interview_exps",
    "detail": {"urls_failed": ["url1", "url2"]}
  },
  "partial_data": { ... }
}
```

## 超时控制
- 单个工作流总超时：60 分钟
- 步骤 1（解析输入）：5 分钟
- 步骤 2（抓取面经）：30 分钟
- 步骤 3（生成报告）：20 分钟
- 步骤 4（保存报告）：1 分钟
